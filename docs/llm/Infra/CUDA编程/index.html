CUDA 编程基础

并行计算 & 异构计算

  • 并行计算：并行计算(parallel computing)是一种计算形式，它将大的问题分解为许多可以并发执行的小问题。并行计算分为：任务并行(task parallel))和数据并行(data parallel))

任务并行指多个任务同时执行

数据并行指多个数据可以同时被处理，每个数据由独立的线程处理

数据并行分块方法：块分(block partitioning)和循环分块(cyclic partitioning)

  • 异构计算(heterogeneous computing)是对并行计算的延伸，是高性能计算发展的里程碑。异构计算指在具有多种类型处理器(CPU和GPU)的系统中完成的计算

CUDA 基础概述​

GPU 指标​

GPU 性能指标：

  • GPU 核心数（core number）
  • GPU 内存容量（显存）
  • 计算峰值：每秒钟单精度或双精度运算能力
  • 内存带宽：每秒钟读出或写入GPU内存的数据量

什么是CUDA？

CUDA（Compute Unified Device Architecture）是英伟达公司推出的基于其GPU的通用高性能计算平台和编程模型。

  1. CUDA C编程时只需要编写顺序执行的程序，在程序代码中不需要有多线程的处理
  2. 要深刻的理解CUDA平台GPU的内存架构和线程架构，并掌握对其控制和优化方法
  3. 掌握CUDA平台的常用性能分析和调优工具：NVIDIA NsightTM、CUDA-GDB、图形化的性能分析工具

系统GPU检测

系统GPU确认：

lspci | gep -i nvidia

判断GPU与CUDA兼容：https://developer.nvidia.com/cuda/gpus

什么是 OEM 显卡？

OEM显卡是指由专业代工厂生产，但最终贴上品牌电脑厂商（如戴尔、惠普、联想）商标，并作为其整机一部分进行销售的显卡。 “OEM”是“Original Equipment Manufacturer”（原始设备制造商）的缩写，意为“代工生产”。

GPU 计算能力（Compute Capability）

GPU计算能力是英伟达公司用以标识GPU硬件版本的数字：MajorVersion.MinorVersion，相同的MajorVersion代表相同的GPU架构。

nvidia-smi --query-gpu=compute_cap --format=csv
1. 次要版本（Minor Version）代表“特性增量”
即便主版本号相同（同架构），次要版本号的不同也意味着硬件功能集的差异。典型例子：
- Ampere架构（主版本8）：8.0（A100加速卡）支持PCIe Gen4和稀疏化训练；8.6（RTX 30系列游戏卡）加入了第二代RT Core；而 8.9（RTX 40系列）虽然主版本没变，但内部新增了Transformer引擎和FP8精度的硬件支持。
- Turing架构（主版本7）：7.0（V100）没有RT Core，而 7.5（RTX 20系列）首次加入了光线追踪硬件单元。

2. 计算能力决定CUDA工具包（CUDA Toolkit）的“生死线”
这是开发中最实用的点：
- NVIDIA在发布新CUDA版本时，会逐步放弃对老架构（老旧主版本）的支持。例如，CUDA 12.x 已不再支持计算能力 3.x（Kepler架构）的显卡。
- 编译CUDA代码时，-arch 参数指定的计算能力必须向下兼容。如果你的显卡是 8.6，编译时指定 -arch=sm_86 性能最佳，但若指定 -arch=sm_89（针对RTX 40系优化），程序在你卡上直接运行会报“Invalid device function”错误。
常见的 MajorVersion.MinorVersion

主版本号 (Major) 对应核心架构，次版本号 (Minor) 代表同架构下的功能增强

主版本号（Major）= 架构的“语言体系”（指令集基线）

这是最硬性的关联。主版本号（Major）直接对应GPU的底层机器指令集（ISA，即指令集架构）。

只要主版本号相同（比如Ampere的8.0和Ada Lovelace的8.9），它们的基础指令集就是兼容的。意味着为Ampere编译的CUDA二进制代码（SASS），理论上可以直接在Ada显卡上运行（通过二进制翻译）。

一旦主版本号改变（如从8跳到9），意味着NVIDIA重写了底层的硬件指令译码器。此时，旧的二进制代码无法直接运行，必须依赖驱动程序进行即时重编译（JIT，即即时编译）才能转译成新架构的指令。

打个比方： 主版本号就像“通用语言”，8代是“英语”，9代是“中文”。文本内容一样，但底层语法和读音完全不同。

次版本号（Minor）= 架构的“功能配置”（硬件加速器增减）

同主版本号下，次版本号代表该架构内部集成的专属硬件单元的不同。例如，主版本8（都是英语）下：

  • 8.0（Ampere数据中心）：有独立的FP64双精度单元，但没有第二代RT Core。
  • 8.6（Ampere游戏卡）：加入了HDMI 2.1和特定的稀疏化计算支持。
  • 8.9（Ada Lovelace）：虽然“口语”还是英语，但内部新增了硬件级FP8 Transformer引擎和着色器执行重排序（SER，即着色器执行重排序）。这些功能在 8.0 和 8.6 的硬件上根本不存在。

因此，次版本号标识的是“同一种语言下，硬件加速菜单的增减”。CUDA运行时库会检查Minor版本，如果程序调用了FP8指令，而显卡是8.6，库就会直接报错“不支持此功能”。
