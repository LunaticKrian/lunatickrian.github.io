在 minikube 上安装 Loki（方案 A：loki-stack 全家桶）

用 grafana/loki-stack Helm chart 一次性部署 Loki + Promtail + Grafana。 本文档为执行步骤说明，所有命令均需你手动在终端运行。

0. 环境信息（已确认）​

项目状态
minikube✅ Running（单节点，12 CPU / ~8 GB 内存）
默认 StorageClass✅ standard（hostpath，动态 PVC，ReclaimPolicy=Delete）
helm✅ v3.21.3
kubectl✅ v1.34.1
工作目录/Users/krian/loki

⚠️ standard 存储类的回收策略是 Delete：删除 release 或 PVC 后数据会丢失。 学习/演示场景没问题，但心里有数即可。

1. 添加 Helm 仓库​

helm repo add grafana https://grafana.github.io/helm-charts
helm repo update

确认仓库已添加：

helm repo list

（可选）查询当前可用版本，便于后续固定版本：

helm search repo grafana/loki-stack --versions

2. 创建命名空间​

kubectl create namespace loki

3. 编写 values 文件​

在 /Users/krian/loki/ 下创建 loki-stack-values.yaml：

# loki-stack-values.yaml
# 开启 Promtail 日志采集 + Grafana 可视化，并给 Loki 挂持久化卷

loki:
  enabled: true
  persistence:
    enabled: true
    accessModes:
      - ReadWriteOnce
    size: 10Gi
    # storageClassName: standard   # 不写则用集群默认 SC（minikube 即 standard）

promtail:
  enabled: true
  # 默认会采集所有命名空间、所有 Pod 的 /var/log

grafana:
  enabled: true
  # 默认账号 admin / admin（首次登录会要求改密码）

用 values 文件的好处：可纳入版本管理，后续改配置直接改文件再 helm upgrade。

4. 安装​

helm install loki grafana/loki-stack \
  --namespace=loki \
  --create-namespace \
  -f loki-stack-values.yaml
# 如需固定版本，追加：  --version <上一步查到的版本号>

说明：--create-namespace 是双保险（第 2 步已建过也无妨）。 release 名字固定为 loki，下文命令都基于这个名字。

5. 验证部署​

# 查看 release
helm list -n loki

# 查看 Pod（等待全部 Running，初次拉镜像可能需要 1~2 分钟）
kubectl get pods -n loki -w

预期看到类似（3 个核心组件）：

loki-0                     1/1     Running
loki-grafana-xxxxxxxx-xxxx 1/1     Running
loki-promtail-xxxxx        1/1     Running

进一步检查：

# 查看 Loki 自动创建的 PVC
kubectl get pvc -n loki

# 查看所有 Service
kubectl get svc -n loki

6. 访问 Grafana​

6.1 获取 admin 初始密码​

kubectl get secret --namespace loki loki-grafana \
  -o jsonpath="{.data.admin-password}" | base64 --decode ; echo

默认用户名 admin，密码为上面命令输出的内容（默认也是 admin）。

6.2 端口转发到本地​

kubectl port-forward --namespace loki svc/loki-grafana 3000:80

浏览器打开：http://localhost:3000

6.3 在 Grafana 中查看日志​

loki-stack 已自动帮你配好：

  • 数据源：Loki（指向集群内的 Loki 服务）
  • Dashboard：内置 "Logs / Apps" 等 dashboard

路径：左侧菜单 → Explore（或 Dashboards）→ 选 Loki 数据源 → 选 {namespace="loki"} 之类的日志流即可查询。

7. 快速自检：确认有日志进来​

# 随便看一个 promtail 容器日志，确认它在采集
kubectl logs -n loki -l app.kubernetes.io/name=promtail --tail=20

或在 Grafana Explore 里执行 LogQL：

{container="coredns"}

能看到 coredns 的日志就说明全链路通了。

8. 升级配置（后续改参数用）​

改完 loki-stack-values.yaml 后：

helm upgrade loki grafana/loki-stack \
  --namespace=loki \
  -f loki-stack-values.yaml

9. 卸载与清理​

# 卸载 release（PVC 会保留，按需手动删）
helm uninstall loki -n loki

# 彻底清理命名空间 + PVC + PV（数据会丢）
kubectl delete namespace loki

10. 常见注意点​

问题说明
Loki Pod 起不来 / 卡在 PVC确认默认 StorageClass 存在（kubectl get sc），minikube 默认有 standard
想看实时日志但没数据检查 promtail Pod 是否 Running；Promtail 采集的是容器 stdout/stderr
Promtail 已进入维护模式仍完全可用，但官方新项目推荐用 Grafana Alloy；若要现代化可参考方案 B
loki-stack chart 较久未更新对学习/演示完全够用；生产建议用官方独立 loki chart（方案 B）
关机后 minikube 重启minikube start 后 PVC 数据仍在（hostpath 存在 minikube 节点磁盘里）

附：一键命令汇总（可逐条复制执行）​

# 3. 观察
kubectl get pods -n loki -w

# 4. 访问 Grafana
kubectl get secret -n loki loki-grafana -o jsonpath="{.data.admin-password}" | base64 --decode; echo
